## 2026-04-06: 生产环境 API 路径解析问题

### 问题描述
生产环境中，前端使用相对路径 `/api` 被错误解析为 `file:///D:/api/` 格式，而非正确的 `http://127.0.0.1:8000/api/`。

### 原因分析
1. **开发环境**：Vite 开发服务器有代理配置，`/api` 会被转发到 `http://127.0.0.1:8000`
2. **生产环境**：Electron 使用 `file://` 协议加载本地 HTML 文件，没有 Vite 代理
3. **webSecurity**：默认的 webSecurity 限制阻止 `file://` 协议访问 `http://` 后端

### 修复方案

#### 1. 前端 API 路径动态检测 ([frontend/src/api.ts](file:///d:/File/User/code/Papyrus/Papyrus_Desktop/frontend/src/api.ts#L1-L4))
```typescript
const BACKEND_URL = 'http://127.0.0.1:8000';
const BASE = window.location.protocol === 'file:' 
  ? `${BACKEND_URL}/api` 
  : '/api';
```

#### 2. Electron 安全配置 ([electron/main.js](file:///d:/File/User/code/Papyrus/Papyrus_Desktop/electron/main.js#L335-L339))
```javascript
webSecurity: true,  // 保持安全保护开启
// allowFileAccessFromFiles 已移除（有安全风险），前端使用绝对路径访问 API
```

#### 3. 后端 CORS 配置 ([src/papyrus_api/main.py](file:///d:/File/User/code/Papyrus/Papyrus_Desktop/src/papyrus_api/main.py#L104-L116))
```python
# 注意：这是本地应用，仅绑定到 127.0.0.1，不暴露到公网
# 使用正则表达式仅允许安全来源：
# - null origin (file:// 协议)
# - http://localhost:*
# - http://127.0.0.1:*
app.add_middleware(
    CORSMiddleware,
    allow_origin_regex="^null$|^http://localhost(:\\d+)?$|^http://127\\.0\\.0\\.1(:\\d+)?$",
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 安全说明
- `webSecurity: true`：保持 Electron 的安全保护措施开启
- 前端使用绝对路径 `http://127.0.0.1:8000/api` 访问后端，无需 `allowFileAccessFromFiles`
- 后端仅绑定到 `127.0.0.1`，不暴露到公网
- CORS 配置使用正则表达式仅允许安全来源（null origin、localhost、127.0.0.1）
