# 📝 笔记自动补全功能 Demo

## 功能概述

基于大模型的智能文本补全功能，在用户编辑笔记时提供上下文感知的续写建议。

## 两种交互模式

### 模式一：实时预览模式（二次确认关闭）

```
用户输入: "今天学习了React的"
          ↓
自动显示灰色补全: "useState和useEffect钩子..."
                      ↑
                    用户按 Tab
                      ↓
            补全被接受，加入正文
```

**特点：**
- 输入时自动显示灰色幽灵文本
- 按 **Tab** 键直接接受补全
- 适合追求效率的快速输入场景

### 模式二：Tab触发模式（二次确认开启）

```
用户输入: "今天学习了React的"
          ↓
用户按 Tab
          ↓
显示灰色补全: "useState和useEffect钩子..."
              ↑
            用户按 Enter
              ↓
    补全被接受，加入正文
```

**特点：**
- 按 **Tab** 键触发显示补全
- 按 **Enter** 键确认接受
- 按 **Esc** 取消补全
- 适合需要精确控制的场景

## 配置选项

在 **设置 → 聊天** 中可以配置：

| 设置项 | 说明 | 默认值 |
|:---|:---|:---|
| 笔记自动补全 | 功能总开关 | 开启 |
| 二次确认模式 | 切换两种交互模式 | 关闭 |
| 触发延迟 | 实时模式的防抖延迟 | 500ms |
| 最大补全长度 | 每次生成的最大token数 | 50 |

## 快捷键

| 按键 | 实时预览模式 | Tab触发模式 |
|:---|:---|:---|
| **Tab** | 接受当前补全 | 触发/接受补全 |
| **Enter** | 正常换行 | 确认接受补全 |
| **Esc** | 取消补全 | 取消补全 |

## 启动测试

1. **启动后端服务：**
```bash
cd c:\Users\li\Documents\GitHub\Papyrus
python -m uvicorn src.papyrus_api.main:app --reload
```

2. **启动前端开发服务器：**
```bash
cd frontend
npm run dev
```

3. **配置 AI：**
   - 打开设置 → 聊天
   - 配置 AI Provider（OpenAI/Ollama 等）
   - 启用笔记自动补全功能

4. **测试补全：**
   - 进入笔记库
   - 新建或编辑笔记
   - 在内容区域输入文字
   - 体验自动补全功能

## 注意事项

1. **AI 配置**：确保已配置有效的 AI Provider，否则补全功能无法使用
2. **网络连接**：云端 API 需要稳定的网络连接
3. **隐私**：本地 Ollama 模型数据不会上传，云端 API 会发送文本内容
